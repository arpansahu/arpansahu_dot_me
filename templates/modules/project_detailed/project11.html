Description

The motivation behind this project was when I started my job search in the Jan month of 2022. All my colleagues were ready to start their new journey in the career for which they were preparing for a long time ago. This motivated me to start my job search, and frankly speaking I am a bit, lazy person. Thus, most tiring and time-consuming task was applying for a job. I have faced a few challenges while my job search is:

  1. Even after feeding python in the search bar for jobs, LinkedIn was also showing jobs without python.
  2. Moreover on the Job Search page after clicking the search button you are shown only the job title and category along with a few details. So for every job you have to check the entire job post that is suitable for you or not.
  3. Job search could be better if job skills would be shown on the main search page already, saving 50% time for job searchers.
  4. Linkedin should make it compulsory to put a range for job posting on how many years of experience is required.
  5. Moreover few companies do not post on LinkedIn. Rather they maintain and update their own careers pages more frequently. Due to this reason, you may lose some good opportunities. To solve this problem you may have to subscribe to their career alerts.
  6. But solving the last problem creates another problem which is a lot of daily emails for jobs which further at some point in time makes it next to impossible to check every email in that detailed manner, leading towards another problem.
and this is how I decided to build my self another job portal platform.
Note: This project is my personal dream project and Repositories are not open for public (like other projects).

This project consists of 4 sub projects


The Admin Panel

  1. Build Admin Panel using Django
  2. Deployed on AWS EC2
  3. Used S3 Route along with Nginx and Gunicorn
  4. OpenSSL used for ssl certificate
  5. Live at: https://technorigger-admin.xyz/

The Cron Instance

  1. This cron instance is built using Django
  2. As of now there are 5 cron commands built for updating Locations, New Jobs, Deleting jobs that are no more available, Associating Skills and Extraction of Skills.
  3. Three companies : Browserstack, Meta and Walmart Global India are being scrapped on a daily basis.
  4. Architecture of Cron Instance with Jobs is almost complete and Now the main focus is on users in Next Instance

The API Instance
This instance is still on Architectural phase. Actually when I started to build it, I wanted it to be future-proof, and I want to use Asynchronous python frameworks such as FAST API so that I can get maximum threshold. Moreover, one more though came into my mind is that, What is the reason which makes LinkedIn a billion-dollar company and Glassdoor not. Its User Interactions. Glassdoor is simply a job posting board or reviews boarding. Although Reviews are being posted by Users only but still it's not that much interactive as of LinkedIn. Because LinkedIn is a social media and Glassdoor is not. Thus, I am thinking to bring neo4j as Graph Database into the picture for building a social network of users.
